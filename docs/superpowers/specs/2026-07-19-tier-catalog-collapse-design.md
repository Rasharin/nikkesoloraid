# 티어 전체 니케 목록 접기/펼치기 설계

## 목표

티어 탭의 `전체 니케 목록` 섹션을 필요할 때 접어 티어 편집 공간을 더 넓게 볼 수 있도록 한다.

## 동작

- 기본 상태는 펼쳐짐이다.
- 펼쳐진 상태에서는 제목, 니케 이름 검색창, 아래 방향 화살표 버튼, 필터, 니케 카드 목록을 표시한다.
- 접힌 상태에서는 `전체 니케 목록` 제목과 위 방향 화살표 버튼만 표시한다.
- 화살표 버튼을 누르면 펼침과 접힘 상태를 전환한다.
- 접었다 다시 펼쳐도 검색어와 선택된 필터는 유지한다.
- 접힘 상태 자체는 저장하지 않으며 페이지를 새로 열면 펼쳐진 상태로 시작한다.

## 컴포넌트 구조

- 변경 범위는 `TierNikkeCatalog` 내부로 제한한다.
- 컴포넌트 내부에 접힘 여부를 나타내는 로컬 상태를 추가한다.
- 제목과 토글 버튼은 항상 렌더링한다.
- 검색창, 필터, 검색 결과 목록 또는 빈 결과 안내는 펼쳐진 상태에서만 렌더링한다.
- 부모 `TierBoard`의 props와 상태 구조는 변경하지 않는다.

## UI와 접근성

- 토글 버튼은 제목 행의 오른쪽, 펼쳐진 상태에서는 검색창 오른쪽에 배치한다.
- 펼쳐진 상태에서는 아래 방향, 접힌 상태에서는 위 방향 화살표를 표시한다.
- 버튼에는 현재 상태를 나타내는 `aria-expanded`를 지정한다.
- 버튼의 접근성 이름과 툴팁은 펼쳐진 상태에서 `전체 니케 목록 접기`, 접힌 상태에서 `전체 니케 목록 펼치기`로 전환한다.
- 기존 테마 변수와 테두리 스타일을 사용해 다크/라이트 모드에 자연스럽게 맞춘다.

## 테스트와 검증

- 소스 회귀 테스트로 기본 펼침 상태, 토글 상태, `aria-expanded`, 검색·필터·목록의 조건부 렌더링을 확인한다.
- 기존 티어 관련 테스트 전체를 실행한다.
- ESLint와 프로덕션 빌드를 실행한다.
- 로그인된 로컬 티어 화면에서 검색창 오른쪽 버튼 위치, 접힘 시 제목과 위 화살표만 남는지, 다시 펼쳤을 때 검색·필터 상태가 유지되는지 확인한다.

## 제외 범위

- 접힘 상태의 로컬 스토리지 저장
- 접기/펼치기 높이 애니메이션
- 부모 컴포넌트나 서버 데이터 구조 변경
