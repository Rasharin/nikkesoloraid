# 티어 간 카드 드래그 오버레이 설계

## 목표

티어 내부 니케 카드를 드래그할 때 현재 티어의 경계를 벗어나도 카드가 사라지지 않고, 포인터를 빠르고 자연스럽게 따라 다른 티어로 이동하도록 한다.

## 원인

현재 티어 카드는 각 티어별 `SortableContext` 안에서 원본 DOM 요소에 변환을 적용한다. 같은 컨텍스트 안의 재정렬에서는 변환이 유지되지만, 다른 티어 컨텍스트로 포인터가 이동하면 원본 요소가 시작 티어에 묶여 있어 드래그 표현이 끊긴다. 행의 `overflow-visible`은 잘림만 방지하므로 컨텍스트 간 드래그 표현의 수명 문제를 해결하지 못한다.

## 선택한 방식

`DndContext` 아래에 항상 마운트된 `DragOverlay`를 추가한다. 티어 카드 드래그가 시작되면 오버레이에 원본과 동일한 카드 표현을 완전 불투명하게 렌더링한다.

- 오버레이는 투명도를 낮추지 않는다.
- 오버레이에는 니케 이미지와 13px 이름 텍스트를 그대로 표시한다.
- 살짝 커지는 크기 변화와 그림자로 카드가 들린 느낌을 준다.
- 원본 카드는 `visibility: hidden`으로 감추되 공간은 유지해, 중복 이미지 없이 재정렬 자리가 밀리는 동작을 보존한다.
- 오버레이는 포인터 이벤트를 받지 않아 기존 충돌 감지와 삽입 위치 계산을 방해하지 않는다.
- 놓을 때 짧은 드롭 애니메이션으로 대상 위치에 연결한다.

## 컴포넌트 구조

`TierBoard.tsx` 안에서 카드의 시각 표현과 드래그 동작을 분리한다.

1. `TierNikkeCardVisual`
   - 이미지와 이름만 렌더링하는 표현 전용 컴포넌트다.
   - 일반 티어 카드와 오버레이가 같은 마크업과 스타일을 공유한다.

2. `TierNikkeCard`
   - 기존 `useSortable` 연결과 접근성 속성을 담당한다.
   - 드래그 중 원본 표현만 숨기고 카드 공간은 유지한다.

3. `DragOverlay`
   - 활성 드래그 소스가 티어일 때만 `TierNikkeCardVisual`을 렌더링한다.
   - 전체 니케 목록에서 시작한 드래그는 현재 동작을 유지하며 별도 오버레이를 만들지 않는다.

## 상태 및 데이터 흐름

드래그 시작 시 활성 카드 ID와 소스를 기존 드래그 상태에 기록한다. 활성 ID로 현재 티어 데이터에서 니케 정보를 조회해 오버레이에 전달한다. 티어를 오가는 동안 기존 `onDragOver`의 삽입 미리보기와 충돌 감지를 그대로 사용한다. 드래그 완료 또는 취소 시 활성 상태가 초기화되고 오버레이 자식이 제거된다.

실제 티어 데이터 저장과 순서 확정은 기존처럼 드래그 종료 시 수행한다. 따라서 시각 개선이 저장 로직이나 권한 처리에 영향을 주지 않는다.

## 예외 처리

- 활성 ID에 해당하는 니케를 찾지 못하면 오버레이를 렌더링하지 않는다.
- 드래그 취소 시 원본 카드의 가시성을 즉시 복원한다.
- 다른 티어의 빈 영역과 카드 사이 모두 기존 충돌 감지 결과를 사용한다.
- 빠르게 여러 티어를 통과해도 오버레이는 `DndContext`에 유지되어 사라지지 않는다.

## 검증

1. 기존의 `DragOverlay` 부재를 요구하는 회귀 테스트를 새 요구사항에 맞게 교체한다.
2. 티어 드래그에만 불투명 오버레이가 사용되는지 소스 테스트로 확인한다.
3. 원본 카드가 드래그 중 공간을 유지하면서 숨겨지는지 확인한다.
4. 전체 테스트, 타입 검사와 프로덕션 빌드를 실행한다.
5. 로컬 브라우저에서 같은 티어 재정렬, 다른 티어 이동, 빈 티어 이동, 드래그 취소를 확인한다.

## 범위 제외

- 전체 니케 목록에서 시작하는 드래그 표현 변경
- 티어 저장 형식이나 데이터베이스 구조 변경
- 충돌 감지 및 삽입 순서 알고리즘의 재설계
