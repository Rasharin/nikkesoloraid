# 티어 전체 초기화 및 화면 경계 리사이즈 설계

## 목표

- 설정의 `전부 초기화`가 티어 데이터뿐 아니라 로컬 화면 크기 설정도 기본값으로 되돌린다.
- 작은 브라우저 창에서 핸들로 섹션을 늘릴 때 섹션이 화면 밖으로 나가지 않게 한다.

## 전체 초기화

`전부 초기화` 확인 후 다음 항목을 한 번에 초기화한다.

- 기본 티어 줄, 이름과 색상
- 모든 니케 배치
- 티어 섹션 가로·세로 크기
- 좌우 이동 위치
- 티어 카드 크기 단계
- 로컬 저장소의 티어 화면 설정

기존 `목록 초기화`와 `화면 크기 설정 초기화`의 개별 동작은 유지한다.

## 화면 경계 제한

- 핸들 드래그를 시작할 때 섹션의 실제 화면 좌표와 브라우저 폭을 측정한다.
- 좌측 핸들은 오른쪽 끝을 고정하고 왼쪽 끝이 화면 왼쪽 16px에 닿으면 너비 증가를 멈춘다.
- 우측 핸들은 왼쪽 끝을 고정하고 오른쪽 끝이 화면 오른쪽 16px에 닿으면 너비 증가를 멈춘다.
- 최소 너비·높이, 세로 조절, 위치 보정과 로컬 저장은 기존 동작을 유지한다.
- 창 크기 변경만으로 저장 크기나 위치를 자동 변경하지 않는다.
- CSS로 넘친 영역을 잘라 숨기지 않고 리사이즈 계산값 자체를 제한한다.

## 구조

- 리사이즈 계산 함수는 선택적인 최대 너비를 받아 최소·최대 범위 안에서 너비를 결정한다.
- `TierBoard`는 드래그 시작 시 핸들 방향별 최대 너비를 계산해 리사이즈 함수에 전달한다.
- `전부 초기화` 콜백은 공용 보드 초기화와 기존 로컬 화면 초기화 함수를 연속 실행한다.

## 검증

- 좌우 핸들이 각각 화면 16px 경계를 넘지 않는지 계산 테스트와 실행 화면에서 확인한다.
- 최대 너비에 닿은 뒤 반대 방향으로 줄이기가 정상 동작하는지 확인한다.
- `전부 초기화` 후 섹션 크기, 위치와 카드 크기가 기본값으로 복원되는지 확인한다.
- 개별 목록 초기화와 화면 크기 초기화 동작에 회귀가 없는지 확인한다.
