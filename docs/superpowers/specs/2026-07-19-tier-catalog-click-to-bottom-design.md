# 전체 니케 목록 이미지 클릭 이동 설계

## 목표

편집 권한자가 전체 니케 목록의 이미지 영역을 단순 클릭하면 해당 니케를 가장 아래 티어의 맨 뒤로 이동한다.

## 동작

- 배치되지 않은 니케는 가장 아래 티어의 맨 뒤에 추가한다.
- 다른 티어에 배치된 니케는 기존 티어에서 완전히 제거한 뒤 가장 아래 티어의 맨 뒤로 이동한다.
- 이미 가장 아래 티어에 있는 니케도 중복 없이 맨 뒤로 재배치한다.
- 이미지 영역의 단순 클릭에만 적용하며 이름 영역 클릭에는 적용하지 않는다.
- 드래그가 시작된 조작은 클릭 이동으로 처리하지 않는다.
- 편집 권한이 없는 사용자는 기존과 같이 목록을 조회만 한다.

## 구조

- `TierNikkeCatalog`는 이미지 클릭 시 니케 이름을 부모로 전달한다.
- `TierBoard`는 마지막 행의 ID를 구하고 기존 `moveNikke()`로 이동한다.
- 기존 `moveNikke()`의 단일 배치 보장 로직을 재사용해 복사나 중복을 방지한다.
- 드래그 활성화 여부를 카드 단위로 추적해 드래그 종료 직후 발생하는 클릭을 무시한다.

## 검증

- 미배치 니케 클릭 시 마지막 행의 맨 뒤에 추가되는지 확인한다.
- 상위 티어 니케 클릭 시 기존 행에서 사라지고 마지막 행에 한 번만 존재하는지 확인한다.
- 마지막 행 니케 클릭 시 중복 없이 맨 뒤로 이동하는지 확인한다.
- 이름 클릭과 드래그에는 클릭 이동이 발생하지 않는지 확인한다.
- 비권한자에게 클릭 이동이 발생하지 않는지 확인한다.
