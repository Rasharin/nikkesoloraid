# 티어 목록 드래그 투명 삽입 공간 설계

## 목표

전체 니케 목록의 카드를 티어 줄로 드래그할 때 나타나는 점선 가이드를 제거한다. 드롭 위치 계산과 주변 카드가 삽입 위치를 비우며 이동하는 동작은 유지해 티어 내부 카드 이동과 같은 흐름을 제공한다.

## 범위

- `TierBoard`의 목록 드래그용 자리표시자에서 테두리와 배경을 제거한다.
- 자리표시자의 너비와 높이는 현재 카드 크기 설정을 그대로 따른다.
- `catalogPreview`, 드롭 대상 행과 인덱스 계산, `DragOverlay`, 드롭 완료 동작은 변경하지 않는다.
- 티어 내부 카드 드래그와 빈 티어 행 표현은 변경하지 않는다.

## 구현 방식

목록 카드의 삽입 위치에 현재와 동일한 크기의 투명 요소를 렌더링한다. 이 요소는 레이아웃 공간만 차지하고 점선, 색상, 배경을 표시하지 않는다. 따라서 드래그 중 다른 카드는 목표 위치를 기준으로 밀리지만 별도의 가이드 도형은 보이지 않는다.

## 검증

- 소스 테스트로 목록 드래그 자리표시자에 점선과 강조 배경 클래스가 없음을 확인한다.
- 기존 목록 삽입 위치 계산 및 드롭 완료 테스트가 계속 통과하는지 확인한다.
- 브라우저에서 목록 카드를 티어 줄의 시작, 중간, 끝으로 드래그해 점선 없이 공간만 열리고 해당 위치에 배치되는지 확인한다.
- 전체 테스트, 관련 ESLint, 프로덕션 빌드를 실행한다.
