# React Effect State 린트 오류 정정 설계

## 목표

`react-hooks/set-state-in-effect` 오류 14개를 기능 변화 없이 제거한다. 저장된 사용자 상태 복원, 문서 편집, 공지 펼침, 덱 선택, 탭 전환 동작은 현재와 동일하게 유지한다.

## 원칙

1. 브라우저 저장소 값은 서버·클라이언트 첫 화면 불일치를 막는 hydration 경계 뒤에서 한 번만 상태에 반영한다.
2. 사용자 동작으로 결정되는 상태는 해당 클릭·전환 이벤트에서 함께 변경한다.
3. props나 다른 상태에서 계산 가능한 값은 렌더링 중 파생하거나 명시적인 재설정 경계로 옮긴다.
4. DOM 관찰처럼 효과가 필요한 경우에는 상태 변경을 관찰자 콜백 안에서 수행한다.
5. 린트 비활성화, 타이머를 이용한 우회, 기능 삭제는 사용하지 않는다.

## 파일별 변경

### `NoticeContent.tsx`

공지 ID 유효성 정리와 첫 공지 자동 펼침을 effect에서 수행하지 않는다. 현재 게시물과 `openIds`를 기준으로 렌더링에 사용할 유효한 펼침 ID 집합을 파생한다. 사용자가 공지를 열고 닫는 상태는 기존 클릭 이벤트가 관리한다.

### `PrivacyContent.tsx`, `TermsContent.tsx`

표시 내용이 바뀔 때마다 draft를 effect로 복사하지 않는다. 편집 시작 버튼을 누르는 순간 최신 `displayText`를 draft에 넣고 편집을 시작한다. 저장 성공 시 편집 종료 동작은 유지한다.

### `HomeTab.tsx`

- 세션의 덱 초안은 전용 읽기 함수로 한 번 파싱하고 hydration 완료 뒤의 조건부 렌더 조정에서 `draft`, `score`, `editingId`에 적용한다.
- 메모도 같은 조건부 렌더 조정에서 localStorage로부터 복원한다.
- 복원 완료 후에만 기존 저장 effect가 실행되어 기본값으로 저장소를 덮어쓰지 않게 한다.
- 외부 `editRequest`의 상태 반영은 이전 요청 ID와 비교하는 조건부 렌더 조정으로 이동한다. 포커스, 알림, 요청 소비 같은 외부 동기화만 effect에 남긴다.

### `ImaginarySoloRaidTab.tsx`

- 저장된 덱 페이지, 활성 페이지, 레이아웃, 추천 영역 열림 여부와 메모는 hydration 완료 뒤의 조건부 렌더 조정에서 각각 안전한 읽기 함수로 복원한다.
- 복원 완료 플래그는 저장값 적용과 함께 설정해 첫 렌더링의 기본값이 저장소를 덮어쓰지 않게 한다.
- 넓은 레이아웃이 꺼졌을 때 높이는 렌더링에서 `null`로 파생하고, `ResizeObserver` 콜백에서만 측정 상태를 갱신한다.
- 선택된 덱 ID는 현재 존재하는 덱 ID와 교집합한 값으로 파생해 렌더링과 동작에 사용한다.

### `UsageTab.tsx`

탭 변경과 작성 시작을 effect로 감지하지 않는다. 탭 변경 래퍼에서 작성창을 닫고, 작성 버튼을 누를 때 현재 게시글을 편집 블록으로 변환한다. 외부에서 전달된 `onTabChange` 호출과 기존 UI 결과는 유지한다.

## 안전한 저장소 처리

hydration 완료 신호는 `useSyncExternalStore`의 서버 스냅샷과 클라이언트 스냅샷 차이를 이용한다. 저장소 복원은 해당 신호가 참이 된 첫 렌더에서 이전 완료 상태와 비교한 조건부 상태 조정으로 한 번만 실행하므로 effect 본문에서 상태를 직접 변경하지 않는다. 모든 읽기 함수는 저장소 접근과 JSON 파싱 실패를 포착하며, 잘못된 값은 기존 기본 상태로 대체한다. 기존 저장 키와 데이터 형식은 변경하지 않는다.

## 검증

1. 14개 오류가 발생하는 파일과 동작을 고정하는 회귀 테스트를 먼저 추가한다.
2. 각 변경 후 관련 테스트를 실행한다.
3. 전체 `npm test`, `npm run lint`, `npm run build`를 실행한다.
4. 로컬 브라우저에서 저장 초안·메모 복원, 문서 편집 시작, 사용법 탭 전환, 공지 펼침, 덱 선택 정리를 확인한다.

## 범위 제외

- 기존 경고 23개의 정리
- 저장 키나 데이터 형식 변경
- 화면 디자인 변경
- 티어 드래그 동작 변경
