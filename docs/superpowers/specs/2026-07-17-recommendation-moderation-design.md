# 추천 조합 데이터 관리 설계

## 목표

허위 점수로 오염된 저장 덱을 마스터가 추천 집계에서 숨기거나 수정·삭제하고, 반복적으로 잘못된 데이터를 올리는 사용자를 추천 집계에서 차단한다.

## 데이터 모델

- `decks.updated_at`: 사용자가 덱을 수정했는지 판단하는 서버 시각.
- `recommendation_deck_moderations`: 덱별 숨김 시각과 당시 덱 수정 시각을 저장한다. 덱의 `updated_at`이 숨김 당시 값보다 새로우면 숨김은 자동 해제된 것으로 간주한다.
- `recommendation_blocked_users`: 차단된 사용자와 차단 처리 마스터, 처리 시각을 저장한다.
- `recommendation_moderation_notices`: 숨김된 덱의 사용자 알림과 확인 시각을 저장한다.

모든 관리 테이블은 RLS를 활성화한다. 마스터 판정은 기존 `app_config.master_user_id`만 사용하며, 클라이언트의 `isMaster` 값은 UI 표시 용도로만 사용한다.

## 서버 인터페이스

- 추천 조회 RPC는 숨김 상태가 유효한 덱과 차단 사용자의 덱을 제외한다.
- 마스터 전용 Route Handler는 조합별 원본 기록 조회, 숨김, 점수 수정, 삭제, 사용자 차단/해제 및 차단 목록 조회를 제공한다.
- 사용자 알림 Route Handler는 로그인 사용자 본인의 미확인 숨김 알림만 반환하고 확인 처리한다.
- 관리 작업 후 추천 캐시를 무효화하고 현재 화면 데이터를 다시 불러온다.

## UI

- 추천 탭에서 마스터에게만 `복사` 오른쪽에 `기록` 버튼을 표시한다.
- 기록 패널은 원본 덱의 점수, 저장 시각, 사용자 식별자, 숨김/차단 상태와 `숨김`, `수정`, `차단`, `삭제` 동작을 제공한다.
- 숨김 알림 문구는 다음으로 고정한다.

  저장된 덱이 현재 서버에 추천 조합에 적용되지 않도록 조치 되었습니다.  
  수정 후 다시 적용이 가능합니다.

- 마이페이지 마스터 관리 영역에 `차단 유저` 섹션을 추가하고 목록 조회와 차단 해제를 제공한다.

## 동작 규칙

- 숨김은 추천의 사용 횟수와 평균 점수에서 해당 원본 덱을 제외한다.
- 덱 소유자가 숨김 이후 점수를 수정하면 다시 추천에 포함한다.
- 마스터 점수 수정은 숨김 기록을 제거하고 교정 점수로 즉시 재집계한다.
- 사용자 차단은 해당 사용자의 기존 및 향후 덱을 모두 제외하며, 덱 저장 자체를 막지는 않는다.
- 차단 해제 시 삭제되지 않은 유효 덱은 다시 추천 대상이 된다.
- 삭제는 확인 후 원본 덱을 영구 삭제한다.
- 요청하지 않은 탭 구조, 인증 저장 방식, 테마, 덱 빌딩, 보스 관리 및 게시판 동작은 변경하지 않는다.

## 검증

- 일반 사용자의 관리 API 접근이 거부된다.
- 숨김·수정·삭제·차단·차단 해제 후 집계가 정확히 갱신된다.
- 숨김 알림은 소유자에게 한 번 표시되고 확인 후 반복되지 않는다.
- 사용자가 숨김 덱을 수정하면 다시 추천에 포함된다.
- 마스터 전용 버튼과 섹션은 일반 사용자에게 표시되지 않는다.
