# 니케 보조 속성 설계

## 목표

기존 니케의 단일 속성을 속성 1로 유지하면서, 일부 니케에 선택적인 속성 2를 저장할 수 있게 한다. 속성 필터는 속성 1과 속성 2 중 하나라도 선택 조건과 일치하면 해당 니케를 포함한다.

## 범위

- 마스터 계정의 니케 등록 폼에서 속성 1과 속성 2를 입력한다.
- 마스터 계정의 니케 목록 및 수정 화면에서 속성 1과 속성 2를 확인하고 수정한다.
- 현재 속성 필터가 있는 모든 니케 목록에서 두 속성을 검색 대상으로 사용한다.
- 카드 표시, 덱 저장 형식, 추천 계산 등 속성 입력과 필터링에 직접 관련되지 않은 동작은 변경하지 않는다.

## 데이터 모델

- 기존 `public.nikkes.element` 열은 이름과 의미를 유지하며 속성 1로 사용한다.
- `public.nikkes.element2 public.element_type null` 열을 추가한다.
- `element2` 단일 열 인덱스를 추가한다. 기존 복합 인덱스는 그대로 유지한다.
- 마이그레이션 시 기존 행의 `element2`는 `NULL`이며, 이는 속성 2 없음으로 해석한다.
- 속성 2가 비어 있거나 속성 1과 같은 경우 저장 직전에 `NULL`로 정규화한다.
- 속성 1은 기존과 같이 니케 등록 시 필수다. 속성 2는 항상 선택 사항이다.

이 구조는 기존 쿼리와 데이터의 호환성을 유지하면서 변경 범위를 최소화한다. 속성 배열이나 별도 관계 테이블은 세 개 이상의 속성을 지원해야 할 때 다시 검토한다.

## 데이터 흐름

1. 전체 니케 조회 쿼리에 `element2`를 포함한다.
2. 클라이언트의 니케 타입과 캐시 정규화 과정에서 `element2`가 없거나 유효하지 않으면 `null`로 취급한다.
3. 마스터 등록 UI가 `element`와 `element2`를 전달한다.
4. 등록 함수가 `element2`를 정규화한 뒤 Supabase에 저장한다.
5. 마스터 수정 UI가 기존 두 속성을 불러와 수정 요청에 포함한다.
6. 수정 함수가 같은 정규화를 거쳐 두 속성을 저장한다.
7. 필터는 선택된 속성 집합에 `element` 또는 `element2`가 포함되면 니케를 표시한다.

## UI

### 니케 등록

- 기존 `속성` 선택 항목의 라벨을 `속성 1`로 바꾼다.
- 같은 행에 `속성 2 (선택)` 선택 항목을 추가한다.
- 등록 완료 후 두 속성 입력값을 모두 초기화한다.
- 속성 2가 속성 1과 같아도 오류를 표시하지 않고 속성 2 없음으로 저장한다.

### 니케 목록 및 수정

- 니케 선택 시 편집 상태에 `element2`를 포함한다.
- `니케 정보 수정`에 기존 속성 항목을 `속성 1`로 표시하고 `속성 2` 항목을 추가한다.
- 속성 2가 없으면 기존 빈 값 표기 방식인 `-`를 사용한다.
- 저장 성공 후 선택된 니케의 로컬 편집 상태에도 정규화된 속성 2 값을 반영한다.

## 필터 동작

각 속성 필터의 기존 다중 선택 방식은 유지한다.

- 선택 속성이 없으면 모든 니케를 표시한다.
- 속성 1이 선택 집합에 있으면 표시한다.
- 속성 2가 선택 집합에 있으면 표시한다.
- 둘 다 선택 집합에 없거나 둘 다 비어 있으면 제외한다.

이 규칙을 니케 관리, 마스터 니케 관리 및 추천 목록, 덱 빌딩의 니케 선택 목록, 티어 니케 목록 등 현재 속성 필터가 존재하는 모든 화면에 동일하게 적용한다.

## 오류 및 호환성 처리

- 이전 캐시에 `element2`가 없어도 `null`로 정규화하여 오류 없이 사용한다.
- DB 마이그레이션이 적용되지 않은 환경에서는 새 열을 포함한 조회와 저장이 실패하므로, 배포 전에 마이그레이션을 먼저 적용한다.
- 속성 1의 기존 필수 검증과 마스터 전용 RLS 정책은 변경하지 않는다.
- 속성 2 때문에 기존 니케 데이터를 일괄 수정하지 않는다.

## 테스트 및 검증

- SQL 마이그레이션에 nullable `element2` 열과 `element2` 단일 열 인덱스가 정의되어 있는지 검사한다.
- 속성 2가 비어 있거나 속성 1과 같을 때 `null`로 정규화되는지 검사한다.
- 속성 1만 일치, 속성 2만 일치, 둘 다 불일치하는 필터 사례를 검사한다.
- 모든 속성 필터 화면이 공통된 두 속성 매칭 규칙을 사용하는지 회귀 검사한다.
- 마스터 등록·수정 UI와 Supabase 조회·저장 payload에 `element2`가 연결되어 있는지 검사한다.
- 전체 테스트, 관련 ESLint, 프로덕션 빌드를 실행한다.
- 실행 중인 앱에서 마스터 등록·수정 UI와 속성 2 필터 결과를 확인한다.
