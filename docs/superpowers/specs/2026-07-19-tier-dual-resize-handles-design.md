# 티어 섹션 양쪽 하단 크기 조절 핸들 설계

## 목표

- 편집 권한자에게 기존 우측 하단 핸들과 대칭인 좌측 하단 핸들을 추가한다.
- 전체 니케 목록의 카드 이미지는 현재보다 한 단계 작게 표시한다.
- 티어 칸에 배치된 카드 크기와 기존 로컬 설정은 유지한다.

## 동작

- 우측 핸들은 현재처럼 오른쪽으로 끌면 너비가 늘어난다.
- 좌측 핸들은 섹션의 오른쪽 끝을 고정한다. 왼쪽으로 끌면 너비가 늘고 오른쪽으로 끌면 줄어든다.
- 두 핸들 모두 아래로 끌면 높이가 늘고 위로 끌면 줄어든다.
- 기존 최소 너비·높이 제한, 로컬 저장, `nwse-resize` 커서와 권한 검사를 유지한다.
- 좌측 핸들의 곡선형 쐐기는 기존 우측 핸들을 수평 반전하며, hover와 dragging 효과도 동일하게 적용한다.

## 구현 범위

- `TierBoard`의 포인터 리사이즈 시작 함수가 `left`와 `right` 방향을 받도록 공용화한다.
- 좌측 리사이즈 중에는 너비 변화량을 반대로 계산하고, 섹션 위치를 함께 보정해 오른쪽 끝이 움직이지 않게 한다.
- 시각적 핸들보다 넓은 포인터 영역을 양쪽에 유지한다.
- `TierNikkeCatalog`의 카드 이미지 크기만 한 단계 줄이고 필터, 검색, 드래그 동작은 유지한다.

## 검증

- 양쪽 핸들의 너비·높이 조절과 최소 크기 제한을 테스트한다.
- 좌측 조절 후 오른쪽 끝 위치가 유지되는지 확인한다.
- 권한이 없을 때 두 핸들이 모두 숨겨지는지 확인한다.
- 전체 니케 목록 이미지만 줄고 티어 칸 카드에는 영향이 없는지 실행 화면에서 확인한다.
