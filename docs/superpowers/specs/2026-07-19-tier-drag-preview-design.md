# 티어 드래그 및 삽입 미리보기 설계

## 목표

- 티어 간 이동 중 잡은 실제 카드가 줄 경계를 넘어도 계속 보이도록 한다.
- 전체 니케 목록의 카드를 티어 카드 사이로 옮길 때 삽입될 공간을 미리 만들고 기존 카드들이 자연스럽게 밀려나도록 한다.

## 티어 간 이동

- 드래그 중인 실제 카드는 원본 이미지, 이름, 배경과 불투명도를 유지한다.
- 티어 줄의 시각적 둥근 테두리는 유지하되 드래그 카드를 자르는 `overflow-hidden` 구조를 제거한다.
- 드래그 카드는 다른 티어 줄 위에서도 가장 앞에 표시된다.
- 별도의 `DragOverlay`, 복제 카드 또는 반투명 이미지는 사용하지 않는다.

## 전체 니케 목록에서 삽입

- 카탈로그 카드를 티어 줄로 가져오면 포인터와 가장 가까운 카드 앞이나 뒤를 삽입 위치로 계산한다.
- 삽입 위치에는 실제 티어 카드와 같은 크기의 빈 공간을 표시한다.
- 기존 티어 카드들은 정렬 애니메이션으로 옆이나 다음 줄로 밀려난다.
- 포인터가 다른 위치로 이동하면 빈 공간도 즉시 해당 위치로 이동한다.
- 드롭하면 미리보기 위치와 동일한 인덱스에 니케를 삽입한다.
- 드래그가 취소되거나 티어 밖으로 나가면 빈 공간을 제거하고 기존 배치로 복원한다.

## 기존 동작 유지

- 티어 줄 전체 드롭 판정을 유지한다.
- 티어에 배치된 니케를 클릭하면 제거되는 동작을 유지한다.
- 같은 티어 안의 순서 변경과 다른 티어로의 이동을 유지한다.
- 이미 배치된 니케는 복사하지 않고 이동한다.

## 검증

- 자동 테스트에서 드래그 카드가 줄 경계에 잘리지 않는 구조인지 확인한다.
- 자동 테스트에서 카탈로그 드래그 중 삽입 위치와 빈 공간이 생성되는지 확인한다.
- 자동 테스트에서 실제 드롭 인덱스가 미리보기 위치와 일치하는지 확인한다.
- 실제 브라우저에서 티어 간 카드가 계속 보이는지, 카탈로그 카드를 카드 사이로 옮길 때 기존 카드가 밀려나는지 확인한다.
