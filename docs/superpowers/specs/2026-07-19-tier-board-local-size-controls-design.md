# 티어 보드 권한자 전용 크기 조절 설계

## 목표

티어 보드 편집 권한자가 자신의 브라우저에서만 티어 섹션의 가로·세로 크기와 니케 카드 크기를 조절할 수 있게 한다. 비권한자에게는 기존 기본 크기를 유지하며, 공용 티어 데이터와 Supabase 스키마는 변경하지 않는다.

## 사용자 동작

### 티어 섹션 크기

- 편집 권한자에게만 티어 섹션 우측 하단에 대각선 형태의 크기 조절 핸들을 표시한다.
- 포인터로 핸들을 드래그하면 섹션의 가로와 세로가 함께 변경된다.
- 현재 기본 렌더링 크기를 최소 크기로 사용하여 그보다 작게 줄일 수 없게 한다.
- 저장된 너비가 현재 화면보다 넓으면 실제 표시 너비는 화면 안으로 제한한다.
- 섹션 높이보다 티어 행의 전체 높이가 크면 행 영역에서 세로 스크롤을 제공한다.

### 니케 카드 크기

- 권한자 전용 티어 설정 패널에 `작게`, `기본`, `크게` 3단계 선택 항목을 추가한다.
- 기존 카드 크기를 `기본`으로 정의한다.
- 크기 단계는 카드 본체, 이미지와 드래그 삽입 자리표시자에 일관되게 적용한다.
- 카드 크기가 달라지면 기존 flex 줄바꿈 배치가 자연스럽게 다시 계산된다.

### 초기화

- 설정 패널에 로컬 크기 설정을 기본값으로 되돌리는 동작을 제공한다.
- 티어 행, 색상, 배치 데이터를 초기화하는 기존 기능과 로컬 크기 초기화는 분리한다.

## 권한과 저장

- 기존 `/api/tier-board` 응답의 `canEdit` 값을 편집 권한의 기준으로 그대로 사용한다.
- 섹션 크기와 카드 크기는 편집 권한이 있을 때만 읽고 적용한다.
- 로컬 설정은 버전이 포함된 전용 `localStorage` 키에 저장한다.
- 저장 값은 너비, 높이, 카드 크기 단계만 포함한다.
- 비권한자는 로컬 설정을 읽거나 적용하지 않고 항상 기존 기본 레이아웃을 본다.
- 티어 보드 API 요청, 공용 보드 데이터 타입, 데이터베이스 구조는 변경하지 않는다.

## 컴포넌트 구조

### `TierBoard`

- 권한자 전용 로컬 레이아웃 상태를 관리한다.
- 최초 렌더링 후 현재 섹션 크기를 측정해 최소 크기와 기본 크기로 확정한다.
- 포인터 드래그 중 임시 크기를 화면에 반영하고 종료 시 로컬 저장소에 저장한다.
- 설정 패널에 카드 크기 값과 변경·초기화 콜백을 전달한다.

### `TierRowView` 및 카드 컴포넌트

- 카드 크기 단계를 전달받아 단계별 크기 클래스를 선택한다.
- 기본 단계의 클래스는 현재 값과 동일하게 유지한다.
- 삽입 자리표시자도 같은 크기 정의를 공유해 드래그 중 레이아웃 이동을 줄인다.

### `TierSettingsPanel`

- 3단계 카드 크기 선택 UI를 표시한다.
- 선택된 단계를 명확히 표시하고 키보드로도 선택 가능하게 한다.
- 로컬 크기 설정 초기화 버튼을 제공한다.

## 반응형 및 접근성

- 크기 조절 핸들은 포인터 조작 영역을 충분히 확보하고 의미 있는 접근성 레이블을 제공한다.
- 모바일처럼 사용 가능한 폭이 저장 너비보다 좁은 경우 `max-width: 100%`로 제한한다.
- 기본 크기 측정 전에는 기존 레이아웃을 그대로 사용해 화면 점프를 최소화한다.
- 카드 크기 선택은 일반 버튼 그룹으로 구현해 키보드와 화면 읽기 도구에서 사용할 수 있게 한다.

## 오류 처리

- `localStorage` 접근 실패, 잘못된 JSON, 범위를 벗어난 숫자, 알 수 없는 카드 단계는 모두 무시하고 기본값을 사용한다.
- 최소 크기 측정값이 없거나 유효하지 않으면 인라인 크기를 적용하지 않는다.
- 로컬 저장 실패는 공용 티어 편집과 자동 저장에 영향을 주지 않는다.

## 검증

- 권한자와 비권한자 렌더링에서 조절 UI 노출 여부를 확인한다.
- 섹션이 초기 크기보다 작아지지 않는지 확인한다.
- 내용이 높이를 넘을 때 내부 세로 스크롤이 생기는지 확인한다.
- 세 카드 단계에서 카드와 삽입 자리표시자의 크기가 함께 바뀌는지 확인한다.
- 새로고침 후 권한자 로컬 설정이 복원되는지 확인한다.
- 손상된 로컬 저장값에서 기본값으로 안전하게 복구되는지 확인한다.
- 기존 티어 이동, 행 설정, 자동 저장 동작에 회귀가 없는지 확인한다.
