# 티어 제거 및 초기화 기능 설계

## 목표

티어에 배치된 니케를 간단히 제거하고, 설정에서 배치만 초기화하거나 티어 섹션 전체를 기본 상태로 되돌릴 수 있게 한다. 선택된 필터 버튼은 덱 빌딩의 `점수 반영` 버튼과 동일한 cyan 계열 스타일을 사용한다.

## 동작

### 티어 니케 클릭 제거

- 편집 권한이 있는 사용자가 티어 줄의 니케 카드를 클릭하면 해당 니케를 모든 티어 줄에서 제거한다.
- 제거된 니케는 전체 니케 목록에는 계속 표시되며 다시 드래그할 수 있다.
- 기존 드래그 이동 동작은 유지한다.
- 열람 전용 사용자의 니케 카드는 클릭해도 변경되지 않는다.

### 초기화

- 설정 패널 제목은 `티어 설정`에서 `설정`으로 변경한다.
- `목록 초기화`는 현재 섹션명, 줄 수, 줄 이름, 순서, 색상을 유지하고 모든 줄의 니케 배치만 비운다.
- `전부 초기화`는 섹션명, 기본 5개 줄, 줄 이름, 순서, 색상, 니케 배치를 `createDefaultTierBoard()` 결과로 복원한다.
- 두 버튼은 실행 전 브라우저 확인창을 표시한다.
- 취소하면 아무 변경도 하지 않는다.
- 실행된 변경은 기존 저장 흐름을 사용한다. 개발 환경에서는 로컬 저장소에만, 운영 환경에서는 기존 권한 및 API 규칙에 따라 저장한다.

### 필터 선택 스타일

- 활성 필터는 덱 빌딩의 `점수 반영` 버튼과 같은 계열인 `border-cyan-500/40 bg-cyan-500/10`을 사용한다.
- 글자는 테마의 기본 텍스트 색상을 사용해 라이트·다크 모드 모두 대비를 유지한다.

## 구성 요소

- `lib/nikke-tier.ts`: 니케 한 명 제거 및 목록 초기화를 순수 함수로 제공한다.
- `TierBoard.tsx`: 카드 클릭을 제거 함수에 연결하고, 전체 초기화 콜백을 설정 패널에 전달한다.
- `TierSettingsPanel.tsx`: 명칭 변경, 두 초기화 버튼, 확인창을 담당한다.
- `TierNikkeCatalog.tsx`: 활성 필터 스타일을 변경한다.

## 검증

- 니케 클릭 제거가 다른 줄과 설정을 변경하지 않는지 테스트한다.
- 목록 초기화가 줄 설정을 유지하면서 니케만 제거하는지 테스트한다.
- 전부 초기화가 기본 5줄과 기본 섹션명으로 복원되는지 테스트한다.
- 읽기 전용 상태에서는 클릭 제거 및 설정 변경 기능이 노출되지 않는지 확인한다.
- 두 초기화 버튼, 설정 명칭, cyan 필터 스타일을 컴포넌트 테스트로 확인한다.
- 전체 테스트, ESLint, 프로덕션 빌드를 실행한다.
