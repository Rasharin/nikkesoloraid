# 니케 티어 탭 설계

## 목표

모든 이용자가 하나의 공용 니케 티어표를 열람하고, 마스터와 기존 기션덱 표기 대상 사용자만 티어표를 편집할 수 있는 `티어` 탭을 추가한다.

## 범위

- 헤더의 `덱 빌딩`과 `사용법` 사이에 `티어` 탭을 추가한다.
- 공개 경로 `/tier`에서 로그인 여부와 관계없이 티어표를 열람할 수 있다.
- 편집자는 마스터 사용자와 기존 기션덱 대상 사용자 UUID `2d455703-52fd-4239-82f8-79c5e1856f30`으로 제한한다.
- 상단에는 티어표, 하단에는 니케 관리 화면과 동일한 형식의 전체 니케 목록·검색·필터를 표시한다.
- 티어표 편집 내용은 공용 데이터로 Supabase에 저장한다.

## 권한 모델

클라이언트는 현재 사용자 UUID와 공개 `app_config.master_user_id`를 비교하고, 기션덱 대상 UUID도 함께 비교하여 편집 UI 노출 여부를 결정한다. 이는 사용성만 위한 판정이며 보안 경계가 아니다.

데이터베이스 RLS는 다음 규칙을 강제한다.

- `anon`, `authenticated`: 티어 보드 조회 가능
- `authenticated`: `auth.uid()`가 `app_config.master_user_id`이거나 기션덱 대상 UUID일 때만 삽입·수정 가능
- 일반 이용자 및 비로그인 이용자: 삽입·수정 불가

기션덱 대상 UUID는 공용 상수로 분리하여 기션덱 조회와 티어 편집 판정에서 같은 값을 사용한다. 사용자 편집 가능한 메타데이터는 권한 판정에 사용하지 않는다.

## 데이터 모델

공용 단일 행 테이블 `public.nikke_tier_board`를 추가한다.

- `id`: 정수 기본키이며 단일 행을 보장하는 값 `1`
- `section_name`: 상단 섹션명, 기본값 `니케 티어`
- `rows`: 티어 줄 배열을 담는 JSONB
- `updated_by`: 마지막 수정 사용자 UUID
- `created_at`, `updated_at`: 생성·수정 시각

각 `rows` 항목은 다음 구조를 사용한다.

```ts
type TierRow = {
  id: string;
  name: string;
  color: string;
  nikkeNames: string[];
};
```

초기 행이 없을 때 앱은 다음 다섯 줄을 표시하며, 최초 저장 시 동일한 값으로 생성한다.

| 이름 | 기본 색상 |
|---|---|
| S | `#ef4444` |
| A | `#f97316` |
| B | `#eab308` |
| C | `#22c55e` |
| D | `#3b82f6` |

저장 전에 데이터는 다음 규칙으로 정규화한다.

- 빈 섹션명과 빈 줄 이름은 직전 값 또는 기본 이름을 유지한다.
- 색상은 `#RRGGBB` 형식만 허용한다.
- 존재하지 않는 니케 이름은 제거한다.
- 한 니케 이름은 전체 줄에 한 번만 허용한다.
- 줄 ID는 중복되지 않는 안정적인 문자열이어야 한다.
- 최소 1줄, 최대 20줄로 제한한다.

## 화면 구성

### 헤더와 라우팅

`TabKey`, `TAB_ROUTE_MAP`, `PATH_TAB_MAP`, 공개 경로 목록에 `tier`와 `/tier`를 추가한다. 헤더는 `덱 빌딩`, `티어`, `사용법` 순서로 표시하고 전체 열 수를 한 칸 늘린다.

### 상단 티어 섹션

상단 카드에는 섹션명과 원형 설정 버튼을 둔다.

- 섹션명: 모든 이용자에게 표시
- 편집자: 섹션명을 더블클릭하면 인라인 입력으로 전환
- 일반 이용자: 텍스트로만 표시
- 설정 버튼: 편집자에게만 표시하며 원형 톱니바퀴 아이콘을 사용

각 티어 줄은 넓은 니케 배치 영역과 우측 이름 영역으로 구성한다.

- 우측 이름 영역은 해당 줄의 지정 색상을 사용한다.
- 줄의 배경과 테두리는 같은 색상을 낮은 투명도로 사용한다.
- 표시 글자색은 색상의 상대 명도를 계산해 검정 또는 흰색으로 자동 결정한다.
- 편집자는 우측 이름을 더블클릭해 인라인으로 수정할 수 있다.
- 배치된 니케 카드는 같은 줄 안에서 순서를 변경할 수 있다.
- 배치된 니케를 다른 줄로 끌면 기존 줄에서 제거되고 새 줄로 이동한다.

### 설정 패널

원형 설정 버튼을 누르면 티어 설정 패널을 연다. 패널은 각 줄을 한 행으로 표시하고 다음 기능을 제공한다.

- 줄 이름 입력
- HTML `input[type="color"]` 기반 색상 선택기와 현재 HEX 값 표시
- 줄 위·아래 이동
- 줄 삭제
- 새 줄 추가

줄 삭제 시 그 줄에 있던 니케는 미배치 상태가 되어 하단 전체 목록에서 다시 드래그할 수 있다. 마지막 한 줄은 삭제할 수 없다. 설정 변경은 화면에 즉시 반영하며 저장 중 상태를 표시한다.

### 하단 전체 니케 목록

니케 관리 탭의 전체 니케 목록과 같은 카드 형식, 검색 위치, 버스트·코드·클래스 필터를 사용한다. 공통으로 재사용할 수 있는 필터링과 카드 표시 단위를 분리하되, 니케 관리의 선택·즐겨찾기 상태는 티어 탭에 영향을 주지 않는다.

- 이름과 별칭 검색
- 버스트 필터
- 코드 필터
- 클래스 필터
- 전체 니케를 항상 목록에 유지
- 이미 티어에 배치된 니케는 배치 위치를 표시하고 다시 드래그할 수 있게 한다
- 편집자가 목록 카드를 티어 줄로 끌면 해당 줄에 추가한다
- 일반 이용자에게는 같은 목록을 표시하되 드래그 핸들과 편집 상호작용은 제공하지 않는다

## 드래그 동작

기존 의존성인 `@dnd-kit/core`와 `@dnd-kit/sortable`을 사용한다. 마우스, 터치, 키보드 센서를 지원한다.

드래그 종료 시 단일 변환 함수가 다음 결과를 계산한다.

1. 전체 티어 줄에서 대상 니케를 제거한다.
2. 드롭한 줄의 지정 위치에 니케를 삽입한다.
3. 같은 줄 안의 드래그라면 순서만 변경한다.
4. 유효한 드롭 대상이 아니면 기존 상태를 유지한다.

따라서 니케는 복제되지 않으며 항상 최대 한 줄에만 존재한다.

## 저장과 오류 처리

초기 진입 시 티어 보드와 니케 목록을 함께 불러온다. 티어 데이터가 없거나 형식이 잘못된 경우 안전한 기본 5줄을 표시한다.

편집은 낙관적으로 화면에 반영한 뒤 짧은 디바운스를 거쳐 저장한다. 저장 요청은 가장 최신 변경만 최종 상태가 되도록 순서를 관리한다.

- 저장 성공: 서버가 반환한 정규화 데이터로 상태 확정
- 권한 오류: 편집 상태를 마지막 서버 상태로 복원하고 권한 안내 표시
- 네트워크/검증 오류: 마지막 서버 상태로 복원하고 재시도 가능한 오류 안내 표시
- 로딩 중: 티어 영역 스켈레톤 또는 로딩 문구 표시

동시에 두 편집자가 수정할 수 있으므로 `updated_at`을 비교하는 낙관적 동시성 검사를 사용한다. 오래된 화면에서 저장하면 덮어쓰지 않고 최신 데이터를 다시 불러오도록 안내한다.

## 구성 요소 경계

- `TierTab`: 데이터 로딩·저장 상태와 전체 화면 조합
- `TierBoard`: 섹션명, 설정 버튼, 티어 줄, 드래그 컨텍스트
- `TierSettingsPanel`: 줄 추가·삭제·순서·이름·색상 설정
- `TierNikkeCatalog`: 검색·필터·전체 니케 카드 목록
- `lib/nikke-tier.ts`: 기본값, 데이터 정규화, 이동 변환, 색상 대비 계산
- `/api/tier-board`: 공개 조회와 권한이 확인된 저장 API

서버 API는 사용자 세션을 쿠키에서 확인하고 쓰기 권한을 검증한다. 데이터베이스 RLS도 동일한 권한을 별도로 강제한다.

## 테스트

### 단위 테스트

- 기본 5줄 생성
- 잘못된 저장 데이터 정규화
- 중복 니케 제거
- 목록에서 티어로 추가
- 티어 간 이동
- 같은 줄 내 순서 변경
- 잘못된 드롭 시 무변경
- 줄 삭제 시 니케 미배치 처리
- 색상 명도에 따른 글자색 선택

### 서버 및 권한 테스트

- 비로그인 공개 조회 성공
- 일반 로그인 사용자 저장 거부
- 마스터 저장 허용
- 기션덱 대상 사용자 저장 허용
- 오래된 `updated_at` 저장 충돌 거부
- RLS 정책이 일반 사용자의 직접 쓰기를 차단

### UI 및 회귀 검증

- 헤더 순서와 `/tier` 라우팅
- 일반 이용자에게 편집 UI가 노출되지 않음
- 편집자의 더블클릭 이름 수정
- 설정 패널의 줄 추가·삭제·정렬·색상 선택
- 마우스·터치·키보드 드래그
- 모바일과 데스크톱 레이아웃
- 기존 니케 관리 탭의 검색·필터·선택 동작이 유지됨

## 제외 범위

- 사용자별 개인 티어표
- 여러 버전의 공용 티어표
- 변경 이력 및 되돌리기
- 서브마스터 관리 화면
- 이미지 업로드 또는 니케 정보 편집
